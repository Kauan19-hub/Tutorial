🚀 COMO SUBIR UM PROJETO NO GITHUB (PASSO A PASSO) 🚀
------------------------------------------------------------

Autor: Kauan Vinícius

Última atualização: 07/08/2025


📌 OBJETIVO:
------------------------------------------------------------
Este guia ensina como subir um projeto local para o GitHub 
pela primeira vez, de forma simples, direta e prática.


Existem duas formas para realizar esse processo. São eles:


1- Usar o prompt de comando; (Avançado)
2- Usar o Terminal - Git Bash do VSCode. (Iniciante)


TUTORIAL USANDO PROMPT DE COMANDO
------------------------------------------------------------


🧰 REQUISITOS:
------------------------------------------------------------
✔️ Conta no GitHub: https://github.com
✔️ Git instalado na máquina: https://git-scm.com
✔️ Terminal (cmd, bash, PowerShell ou outro)
✔️ Projeto criado localmente (pasta com seus arquivos)


📚 PASSOS:
------------------------------------------------------------


✅ INICIALIZE UM REPOSITÓRIO LOCAL
------------------------------------------------------------
Abra o terminal na pasta do seu projeto e execute:


    git init


Isso cria uma pasta oculta `.git/` com as configurações do repositório.
------------------------------------------------------------


📂 ADICIONE OS ARQUIVOS PARA COMMIT
------------------------------------------------------------
Adicione todos os arquivos do projeto para serem "commitados":


    git add .


Você também pode adicionar arquivos específicos:


    git add nome_do_arquivo.extensao
------------------------------------------------------------


📝 FAÇA O PRIMEIRO COMMIT
------------------------------------------------------------
Agora salve o estado atual do projeto com uma mensagem:


    git commit -m "Primeiro commit"


Dica: use mensagens curtas e claras.
------------------------------------------------------------


🌐 CRIE UM REPOSITÓRIO NO GITHUB
------------------------------------------------------------
- Vá até https://github.com
- Clique no botão ➕ "New Repository"
- Escolha um nome (ex: meu-projeto)
- (Opcional) Adicione descrição, README, .gitignore
- Clique em "Create Repository"


⚠️ **IMPORTANTE**: Se você já criou README/.gitignore lá, precisará clonar o repositório ou fazer um `pull` antes do `push`
------------------------------------------------------------


🔗 VINCULE O REPOSITÓRIO REMOTO
------------------------------------------------------------
Copie o link HTTPS do seu repositório (ex: https://github.com/seuusuario/meu-projeto.git)


No terminal, vincule seu projeto local ao repositório remoto:


    git remote add origin https://github.com/seuusuario/meu-projeto.git


Você pode verificar com:


    git remote -v
------------------------------------------------------------


⬆️ ENVIE OS ARQUIVOS PARA O GITHUB
------------------------------------------------------------
Agora envie seus commits locais para o GitHub:


    git push -u origin master


Se estiver usando `main` ao invés de `master` (mais comum):


    git push -u origin main


⚠️ O terminal pode pedir login/senha ou token do GitHub.
------------------------------------------------------------


✅ CONFIRA SE DEU CERTO
------------------------------------------------------------
Abra seu navegador e acesse o repositório no GitHub.
Se tudo deu certo, os arquivos estarão lá!


📌 COMANDOS ÚTEIS FUTUROS:
------------------------------------------------------------


🔄 Verificar status do repositório:
    git status


📜 Ver histórico de commits:
    git log


🆕 Adicionar novos arquivos e subir:
    git add .
    git commit -m "Nova atualização"
    git push

TUTORIAL USANDO TERMINAL - GIT BASH - VSCODE
—-----------------------------------------------------------


🧰 REQUISITOS:
------------------------------------------------------------


✔️  Ter o Git instalado → https://git-scm.com
✔️  Ter uma conta no GitHub → https://github.com
✔️  Ter o VS Code instalado → https://code.visualstudio.com/
✔️  Git Bash instalado (vem junto com o Git)
✔️  Já ter um projeto criado em alguma pasta do seu PC


📂 ADICIONE OS ARQUIVOS PARA COMMIT
------------------------------------------------------------
Adicione todos os arquivos do projeto para serem "commitados":


    git add .


Você também pode adicionar arquivos específicos:


    git add nome-do-arquivo.ext


Exemplo:


    git add index.html style.css
------------------------------------------------------------


💬 FAÇA UM COMMIT COM UMA MENSAGEM
------------------------------------------------------------
O commit salva um "estado" atual dos arquivos no repositório local:


    git commit -m "mensagem explicando o que foi feito"


Exemplo:


    git commit -m "Subindo arquivos iniciais do projeto"
------------------------------------------------------------


🌐 CRIE UM REPOSITÓRIO NO GITHUB
------------------------------------------------------------
- Acesse: https://github.com
- Clique no botão `+` no canto superior direito → `New repository`
- Escolha um nome para o repositório (pode ser o mesmo da pasta)
- NÃO marque nenhuma opção de README, .gitignore ou License (deixe vazio)
- Clique em `Create repository`
------------------------------------------------------------


🔗 CONECTE O REPOSITÓRIO LOCAL AO GITHUB
------------------------------------------------------------
Copie a URL do repositório que você acabou de criar (formato HTTPS):


Exemplo:  
`https://github.com/seu-usuario/nome-do-repo.git`


No terminal (na pasta do seu projeto), execute:


    git remote add origin https://github.com/seu-usuario/nome-do-repo.git
------------------------------------------------------------


📤 ENVIE OS ARQUIVOS PARA O GITHUB
------------------------------------------------------------
Primeiro, defina o nome do branch principal como `main`:


    git branch -M main


Agora, envie os arquivos para o GitHub:


    git push -u origin main


Se for a primeira vez, o Git pode pedir:
- seu **nome de usuário**
- e seu **Token de Acesso Pessoal (PAT)** no lugar da senha


Você pode gerar um token aqui:
👉 https://github.com/settings/tokens
------------------------------------------------------------


🎉 PRONTO! SEU PROJETO ESTÁ NO GITHUB! 🎉 
------------------------------------------------------------
Sempre que quiser atualizar seu repositório com novas mudanças:


    git add .
    git commit -m "sua mensagem"
    git push
------------------------------------------------------------


💎RESUMINDO:
------------------------------------------------------------
- git init
Inicia a branch;


- git status
Verifica se existe arquivos adicionados;


- git add .
Adiciona arquivos da pasta;


- git status
Verifica os arquivos adicionados;


- git commit -m “nome do repositório”
Define o nome do repositório;


- git branch -M master (ou main)
Define Master, ou Main;


- git remote add origin https://…. (link do repositório)
Adiciona os arquivos no repositório;


- git push -u origin master (ou main)
Puxa os arquivos para o repositório.
